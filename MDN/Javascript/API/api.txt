APIs(Application Programming Interfaces)
    - JavaScript : 웹 페이지/ 앱에서 기능을 구현할 수 있도록 브라우저에 내장 된 고급 스크립팅 언어
    - 브라우저 API : JavaScript 언어 위에 있는 브라우저에 내장 된 구성으로 기능을 보다 쉽게 구현
    - 타사 API : 제 3자 플랫폼에 내장 된 구성으로, 자신의 웹 페이지에서 이러한 플랫폼의 일부 기능을 사용할 수 있음
    - JavaScript 라이브러리 : 웹 페이지에 첨부하여 공통 기능 작성 속도를 높이거나 활성화 할 수 잇는 사용자 정의 함수가 포함 된 하나 이상의 JavaScript 파일
        ( ex) jQuery, Mootools 및 React)
    - JavaScript 프레임워크  
        - HTML, CSS, JavaScript 및 기타 기술의 패키지
        - 처음부터 전체 웹 애플리케이션을 작성하는 데 사용
        - 라이브러리와 프레임워크의 주요 차이점은 제어를 누가 하느냐
            - 라이브러리 : 개발자
            - 프레임 워크 : 반전
    
Web browser
    - window : 웹 페이지가 로드되는 브라우저 탭
        - window 객체에 의해 JavaScript로 표현
    - navigator : 웹에 존재하는 브라우저의 상태와 신원을 나타냄
        - 사용자가 선호하는 언어, 사용자 웹캠의 미디어 스트림 등을 검색 할 수 있음
    - document : window에 로드 된 실제 페이지
        - 문서를 구성하는 HTML 및 CSS에 대한 정보를 반환하고 조작할 수 잇음
    - dom 트리
        - 요소 노드 : DOM에 있는 요소
        - 루트 노드 : HTML의 경우 항상 HTML 노드인 트리의 최상위 노드
        - 자식 노드 : 다른 노드 바로 안에 있는 노드
        - 후손 노드 : 노드 어디서나 노드 내부.
        - 부모 노드 : 내부에 다른 노드가 있는 노드 
        - 형제 노드 : DOM 트리에서 동일한 수준에 있는 노드 
        - 텍스트 노드 : 텍스트 문자열을 포함하는 노드

CSS 스타일 조작
    - 동적으로 스타일을 지정하려는 요소에 직접 인라인 스타일을 추가하는 것
        - HTMLElement.style 문서의 각 요소에 대한 인라인 스타일 정보를 포함 하는 속성으로 수행
        
서버에서 데이터 가져오기
    - 서버에서 개별 데이터 항목을 검색하여 전체 새 페이지를 로드하지 않고도 웹 페이지의 섹션을 업데이트 하는 것
    - 페이지의 일부를 업데이트 할 때마다 전체 페이지를 다시 로드하는 것을 개선
    - AJAX
        - 웹 페이지에서 작은 데이터 청크를 요청하고 필요할 때만 표시함
        - XMLHttpRequest의 Fetch API를 사용
        - 이러한 기술을 통해 웹 페이지는 서버에서 사용 가능한 특정 리소스에 대한 HTTP 요청을 직접 처리하고 결과 데이터가 표시되기 전에 필요에 따라 형식을 지정할 수 있음
    - XMLHttpRequest
        - ajax-start.html

비디오 및 오디오 API
    - HTML5 비디오 및 오디오
    - 네이티브 컨트롤은 브라우저마다 다르기떄문에 브라우저 간 지원에 안좋고, 키보드로 엑세스 할 수 없음
      => 네이티브 컨트롤을 숩기고 직접 프로그래밍 하여 이러한 문제를 해결

Client-side storage
    - 개발자가 클라이언트 측에 데이터를 저장할 수 있고 필요할 때 가져올 수 있게 해주는 자바스크립트 API
    - 웹사이트에 대한 선호를 개인화하기(사용자가 선택한 커스텀 위젯, 배색, 폰트 크기로 보여주기)
    - 이전 활동 기록 저장하기(이전 세션에 담았던 장바구니 목록 저장하기, 로그인 유지하기)
    - 사이트 다운로드가 빨라지고 네트워크 연결 없이도 사용할 수 있게끔 데이터를 로컬에 저장하기
    - 오프라인 상태에서 사용할 수 있도록 웹 어플리케이션 생성 문서를 로컬에 저장하기
    - 클라이언트-사이드 저장소와 서버-사이드 저장소는 대게 함께 사용됨
      ex) 음악 파일 여러 개를 다운받아 클라이언트-사이드 데이터베이스에 저장하고 필요할 때 재생할 수 있음
    
    - cookies(old)
        - 웹에서 보편적으로 사용된 클라이언트-사이드 저장소의 제일 오래된 형태
        - cookies는 세션 ID나 access token 같은 사용자 상태와 개인화에 관련된 정보를 저장하는 여전히 보편적으로 쓰임
    - Web storage and indexedDB(new)
        - Web Storage API  
            - 이름과 대응되는 값으로 이루어진 더 작은 데이터를 저장하고 가져오는 매우 간단한 기능을 제공
            - 사용자 이름과 로그인 여부, 스크린 배경색을 어떤 색으로 할지 등등 같은 간단한 데이터를 저장할 필요가 있을때 유용
        - indexedDB API 
            - 복잡한 데이터를 저장할 수 잇는 오나벽한 데이터베이스를 브라우저에서 제공할 수 있게 해줌
            - 소비자 기록의 복잡한 데이터셋부터 오디오나 비디오 파일 같은 더욱 복잡한 데이터까지 저장하는데 쓰일 수 있음
    - Cache (future)
        - 특정 requests에 대한 HTTP responses를 저장하기 위해 디자인
        - 웹사이트가 차후에 네트워크 연결 없이도 사용될 수 있도록 사이트 정보를 저장하는 등의 일을 하는데 유용

    - 각 도메인에 대한 별도의 스토리지
        - 웹 사이트가 서로의 데이터를 볼 수 있으면 보안 문제가 발생
        - 각 도메인에 대해 별도의 데이터 저장소가 있음
