APIs(Application Programming Interfaces)
    - JavaScript : 웹 페이지/ 앱에서 기능을 구현할 수 있도록 브라우저에 내장 된 고급 스크립팅 언어
    - 브라우저 API : JavaScript 언어 위에 있는 브라우저에 내장 된 구성으로 기능을 보다 쉽게 구현
    - 타사 API : 제 3자 플랫폼에 내장 된 구성으로, 자신의 웹 페이지에서 이러한 플랫폼의 일부 기능을 사용할 수 있음
    - JavaScript 라이브러리 : 웹 페이지에 첨부하여 공통 기능 작성 속도를 높이거나 활성화 할 수 잇는 사용자 정의 함수가 포함 된 하나 이상의 JavaScript 파일
        ( ex) jQuery, Mootools 및 React)
    - JavaScript 프레임워크  
        - HTML, CSS, JavaScript 및 기타 기술의 패키지
        - 처음부터 전체 웹 애플리케이션을 작성하는 데 사용
        - 라이브러리와 프레임워크의 주요 차이점은 제어를 누가 하느냐
            - 라이브러리 : 개발자
            - 프레임 워크 : 반전
    
Web browser
    - window : 웹 페이지가 로드되는 브라우저 탭
        - window 객체에 의해 JavaScript로 표현
    - navigator : 웹에 존재하는 브라우저의 상태와 신원을 나타냄
        - 사용자가 선호하는 언어, 사용자 웹캠의 미디어 스트림 등을 검색 할 수 있음
    - document : window에 로드 된 실제 페이지
        - 문서를 구성하는 HTML 및 CSS에 대한 정보를 반환하고 조작할 수 잇음
    - dom 트리
        - 요소 노드 : DOM에 있는 요소
        - 루트 노드 : HTML의 경우 항상 HTML 노드인 트리의 최상위 노드
        - 자식 노드 : 다른 노드 바로 안에 있는 노드
        - 후손 노드 : 노드 어디서나 노드 내부.
        - 부모 노드 : 내부에 다른 노드가 있는 노드 
        - 형제 노드 : DOM 트리에서 동일한 수준에 있는 노드 
        - 텍스트 노드 : 텍스트 문자열을 포함하는 노드
